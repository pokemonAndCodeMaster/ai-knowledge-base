## 🧭 一、什么是 ai-librarian Skill？

`ai-librarian` 是运行在 AI 助手（Antigravity）上的一个**专用角色扩展模块**（Skill）。它将 AI 从"通用聊天机器人"转变为一名严格执行 **三层 Zettelkasten 架构** 的 "数字图书管理员"。

**核心分工**：
- **你（总编辑）**：决定读什么、验证深层逻辑、审批 Lint 报告；
- **AI（图书管理员）**：负责一切苦力——摄入、拆解、写卡、更新索引、维护日志、填写双向链接。

技能文件位于：`~/.gemini/antigravity/skills/ai-librarian/SKILL.md`

---

## 📋 二、三大核心操作 + Prompt 模板

### 操作 1：Ingest（摄入新知识）

**用途**：将一篇文章、书章节、推文、或你的个人见解，正式补充进知识库。

**触发 Prompt 模板**：
```
[Ingest] 请把以下内容摄入到我的知识库（llm_wiki）中：

<内容开始>
[在这里粘贴原文、或给出文件路径如 raw/xxx.md]
<内容结束>

要求：
1. 提炼核心思维模型（别给我全文摘要）；
2. 为每个新概念各建一张原子中文 wiki 卡片，保存到 wiki/ 目录；
3. 检查与已有卡片的关联，并用 [[双向链接]] 标注支持/反驳关系；
4. 更新 index.md 的目录条目；
5. 在 log.md 中追加本次操作日志。
```

**简化触发短语**（AI 已训练识别）：
- "把这个摄入到知识库"
- "Ingest 这篇文章"
- "把这个知识点归档进去"

---

### 操作 2：Query + Dual-Output（查询与双重输出）

**用途**：向 AI 提问知识深度综合问题，并强制要求它把有价值的答案写进 wiki。

**触发 Prompt 模板**：
```
[Query + 双重输出] 请调取 wiki/ 里的相关知识回答我的问题，
并把你的回答整理成一张新的 wiki 卡片存进去。

我的问题是：
[在这里写下你的问题]
```

**关键**：`双重输出法则` 意味着你的问题激发的洞察不会消失在聊天历史中。每一次有价值的 Query 都会自动沉积为一张新知识卡片。

**简化触发短语**：
- "查一下 wiki 里关于 XXX 的内容，并把答案存下来"
- "结合知识库帮我分析 XXX，双重输出"

---

### 操作 3：Lint（知识库体检）

**用途**：定期检查知识库的健康状况，找出孤岛、矛盾、过时内容。

**触发 Prompt 模板**：
```
[Lint] 请对当前 wiki/ 目录执行一次完整体检，生成中文健康检查报告。
重点检查：
1. 是否有孤岛页面（没有被任何其他卡片链接的孤立卡片）；
2. 是否有概念矛盾（不同卡片关于同一概念说法不一）；
3. 是否有在 index.md 中提到但没有对应文件的"幽灵链接"；
4. 是否有重要概念被多处提及但仍无专属卡片。

不要擅自修改文件，等我批准后再动。
```

**简化触发短语**：
- "对知识库做个体检"
- "Lint 一下 wiki"
- "检查一下知识库有没有问题"

## 🔑 四、使用此 Skill 的最佳实践

| 场景 | 推荐做法 |
|---|---|
| **刚看完一篇文章** | `[Ingest] 把这个摄入进去` + 粘贴内容 |
| **想深问某个概念** | `[Query+双重输出]` + 你的问题 |
| **攒了一堆资料** | 先用 `nlm source content` 批量导出到 `raw/`，再分批 Ingest |
| **周末知识维护** | `[Lint]` 体检 → 审批报告 → 补链接 |
| **不确定用哪种操作** | 直接描述你的需求，AI 会判断对应哪种操作 |

## 🚦 五、认知注意事项

- **Skill ≠ 万能**：ai-librarian 只能在加载了此 Skill 的 AI 会话中生效。每次新开对话需要 AI 自动识别（触发词："摄入"/"ingest"/"体检"/"lint"/"知识库"）。
- **原子性原则**：卡片不要太长，一张卡只记一个核心概念，复杂内容要拆开建卡并互相链接。
- **你是总编辑**：Lint 报告出来之前不要轻易信任 AI 的独立判断，最终的删除/合并/重构权在人类手中。
