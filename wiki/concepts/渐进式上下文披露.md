# 渐进式上下文披露

**关联节点**：[[Agent Harness Engineering]] | [[AGENTS.md 标准]] | [[Context Reset vs Compaction]] | [[Harness Stack 分层架构]]

---

## 定义

渐进式上下文披露是让 agent 在需要时**跨层递归地发现上下文**，而不是在开始时将所有信息一次性加载进上下文窗口。

---

## 核心原则

> "AGENTS.md 是目录，不是百科全书。指向更深的真相来源；不要把所有东西都堆在一个文件里。"

---

## 实现机制

```
AGENTS.md（顶层目录）
    ↓ 指向
skills/ 目录（专题技能）
    ↓ 指向
具体文档 / 规则文件（详细内容）
    ↓ 按需加载
```

Agent 首先读取 AGENTS.md 获取概览，然后根据当前任务按需深入阅读相关 skill 文件和具体文档。

---

## 对比：一次性加载 vs 渐进式

| 方式 | 问题 | 渐进式优势 |
|---|---|---|
| 一次性加载所有内容 | 上下文窗口溢出，关键信息被淹没 | 只在需要时加载相关内容 |
| 超长系统提示 | Agent 难以从中检索关键信息 | 分层组织，易于导航 |
| 静态固定提示 | 随项目演化需要频繁全量更新 | 各层独立更新 |

---

## Claude Code 的实践

Claude Code 团队在 agent 工具设计上的核心原则之一：
- 给 agent 一个 todo list 工具 → 后来发现模型变强后工具反而成了约束
- **结论**：随着模型改进，定期去除不再需要的组件，这也是渐进式披露的内核
