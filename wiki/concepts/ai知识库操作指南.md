# AI 知识库操作指南：如何触发 ai-librarian Skill

**关联节点**：[[notebooklm_mcp用法]] | [[统一学习与知识管理框架]] | [[复利知识库]]

## 🧭 一、什么是 ai-librarian Skill？

`ai-librarian` 是运行在 AI 助手（Antigravity）上的专用角色扩展模块。它将 AI 从"通用聊天机器人"转变为一名严格执行 **Karpathy LLM Wiki 三层架构** 的"数字图书管理员"。

**核心分工**：
- **你（总编辑）**：决定读什么、验证深层逻辑、审批 Lint 报告
- **AI（图书管理员）**：承担所有苦力——摄入、拆解、写卡、更新索引、维护日志、填写双向链接

**Skill 文件路径**：`~/.gemini/antigravity/skills/ai-librarian/SKILL.md`

---

## 📋 二、三大核心操作与 Prompt 模板

### 操作 1：Ingest（摄入新知识）

**用途**：将文章、见解、raw/ 中的材料，正式编译进 wiki/。

**标准 Prompt 模板**：
```
[Ingest] 请把以下内容摄入到我的知识库（llm_wiki）：

<内容开始>
[粘贴原文 或 给出文件路径，如 raw/articles/xxx.md]
<内容结束>

要求：
1. 在 wiki/sources/ 创建该资料的精炼摘要页；
2. 为新概念/实体在 wiki/concepts/ 或 wiki/entities/ 创建原子卡片；
3. 用 [[双向链接]] 标注与已有卡片的支持/反驳关系；
4. 更新 index.md 的目录条目；
5. 在 log.md 追加本次操作日志。
所有输出使用中文。
```

**简化触发短语**（AI 会自动识别）：
- `"把这个摄入到知识库"`
- `"Ingest 这篇文章"`
- `"帮我归档这个"`
- `"把这个加进 wiki"`

---

### 操作 2：Query + Dual-Output（查询与双重输出）

**用途**：向 AI 提深度综合问题，并强制要求把有价值的分析存入 wiki/synthesis/。

**标准 Prompt 模板**：
```
[Query + 双重输出] 请调取 wiki/ 里的相关知识来回答以下问题，
并把你的分析整理成一张新的 wiki 卡片存入 wiki/synthesis/。

问题：
[在这里写下你的问题]
```

**核心价值**：`双重输出法则` 确保每次有价值的 Query 都会自动沉积为一张永久知识卡，永不消失在聊天历史中。

**简化触发短语**：
- `"查一下 wiki 里关于 XXX 的内容，并把答案存下来"`
- `"结合知识库分析 XXX，双重输出"`

---

### 操作 3：Lint（知识库体检）

**用途**：定期检查知识库健康状况。建议每周或积累 10+ 张新卡后执行一次。

**标准 Prompt 模板**：
```
[Lint] 请对当前 wiki/ 执行完整体检，生成中文健康检查报告。
重点检查：
1. 🔴 矛盾：不同卡片对同一事实描述冲突；
2. 🟡 过时：被新来源推翻的旧观点（标注 status: stale）；
3. ⚪ 孤岛：无任何入站链接的孤立页面；
4. 🔵 缺页：被 [[引用]] 但没有对应文件的概念；
5. 🟢 建议：推荐值得进一步研究的方向。
不要擅自修改文件，等我批准后再动。
```

**简化触发短语**：
- `"对知识库做个体检"`
- `"Lint 一下 wiki"`
- `"检查一下知识库有没有问题"`

## 🔑 四、最佳实践速查

| 场景 | 推荐做法 |
|---|---|
| **刚看完一篇文章** | `[Ingest]` + 粘贴内容或给出 raw/ 路径 |
| **想深问某个概念** | `[Query + 双重输出]` + 问题 |
| **积累了一批 raw/ 材料** | 用 `nlm source content` 导出后，逐批 Ingest |
| **周末知识维护** | `[Lint]` 体检 → 审批报告 → 确认修复 |
| **不确定用哪种操作** | 直接描述需求，AI 会判断对应操作类型 |
| **想查看知识库概况** | 读 `index.md`，或用 `qmd query "关键词"` |

## 🚦 五、注意事项

- **Skill 触发机制**：ai-librarian 通过触发词自动激活（`摄入/ingest/体检/lint/知识库`），每次新开对话都会重新识别。
- **原子性原则**：卡片太长时主动拆分。一个新概念 = 一张新卡片，复杂内容用 [[链接]] 串联而非合并。
- **你是总编辑**：Lint/重构的最终决定权在你手中，AI 只提建议不擅自大幅改动。
- **Frontmatter 必填**：每张新卡必须包含 `title, tags, created, updated, sources, status` 字段。
