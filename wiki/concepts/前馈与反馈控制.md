# 前馈与反馈控制

**关联节点**：[[Agent Harness Engineering]] | [[赛博控制论与Harness]] | [[martin_fowler_harness_engineering]] | [[可维护性Harness]]

---

## 两种控制类型

| 类型 | 名称 | 触发时机 | 目的 |
|---|---|---|---|
| **Feedforward** | Guides（引导） | Agent 行动**前** | 增加首次产出好结果的概率 |
| **Feedback** | Sensors（传感器） | Agent 行动**后** | 帮助 agent 自我纠正 |

> 只有反馈而没有前馈 → agent 反复犯同样的错误
> 只有前馈而没有反馈 → agent 编码规则但永远不知道是否有效

---

## 两种执行类型

| 类型 | 执行者 | 速度 | 可靠性 | 典型示例 |
|---|---|---|---|---|
| **计算型（Computational）** | CPU | 毫秒~秒 | 确定性 | 测试、linters、类型检查、结构分析 |
| **推理型（Inferential）** | GPU/NPU | 慢且贵 | 非确定性 | AI 代码审查、LLM as judge |

计算型传感器便宜到可以在每次变更时运行；推理型传感器用于计算型无法判断的语义问题。

---

## 时机：质量左移（Keep Quality Left）

```
提交前         │ 集成后         │ 持续监控
快速检查       │ 深度检查       │ 漂移监控
linters        │ 变异测试       │ 死代码检测
快速测试套件   │ 全面代码审查   │ 依赖扫描
基础代码审查   │               │ 运行时 SLO 监控
```

越早发现问题，修复成本越低。

---

## 实际应用（Stripe Minions 案例）

Stripe 的 "Minions" 系统中：
- **预推送钩子**（pre-push hooks）：基于启发式运行相关 linters → 前馈引导
- **蓝图（blueprints）**：将反馈传感器集成进 agent 工作流 → 反馈控制
- 核心原则：将反馈尽可能左移
