It requires the Dataview plugin to function.\n\nThe feature added by this plugin is the capability of automatically serializing dataview queries to Markdown.\n\nWhen you write a normal Dataview query, \"all you get\" is a live view showing the results of the query. The output is never saved, it's just refreshed every time you look at the note containing the query. If you open that note with another editor, all you can see if the query, not the results.\n\nWith the Dataview Serializer plugin, you not only see the query, but also the output. This is powerful because it means that the data IS there. If you publish the note (e.g., using Obsidian Publish, Quartz or GitHub pages), then the data will also be there. As an added benefit, the data may also appear on your graph, which is mighty powerful.\n\nHere's what it looks like:\n\n<!-- QueryToSerialize: LIST FROM #quotes WHERE public_note = true SORT file.name ASC -->\n<!-- SerializedQuery: LIST FROM #quotes WHERE public_note = true SORT file.name ASC -->\n- [[20 years from now, the only people who will remember that you worked late are your kids]]\n- [[A beautiful book is a victory won in all the battlefields of human thought.md|A beautiful book is a victory won in all the battlefields of human thought]]\n- [[A busy mind accelerates the perceived passage of time.
